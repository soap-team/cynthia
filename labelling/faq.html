Cynthia: diff labelling interface

Instructions

Thank you for helping us with either labelling or reviewing edits. We will use this data to improve and develop our vandalism detection tools. The majority of people are going to be here to review edits - please read the instructions about 'reviewing' edits.

If you are a reviewer, you should only label edits in 'cynthia-v3-review'. This is the feed that will be used when we start cleaning up edits automatically.

I am here to review

There are two buttons you can apply to any edit.

  • False positive (F) This is an edit that should not have been reverted by Cynthia. Pressing this button will send a message into the #cleanup-review channel.
  • Save and next: This is an edit that is vandalism.

I am here to label

There are three labels that you can apply to any edit.

  • Good edits should have no labels. There is no need to label a good edit 'good-faith'.
  • Any edit the VSTF can handle should be labelled as damaging or spam.
  • Damaging: Does the edit deliberately remove or damage the usefulness of the article content? Is it vandalism?
  • Spam: Is the edit an unwanted advertisement?
  • Good-faith: Does it appear as though the author of this edit was trying to contribute productively? Sometimes, a revert is made by accident and adds bad content, or a user accidentally removes a word while making good edits.

Frequently asked questions

1. I am unsure of an edit

Choose the best labels that describe the edit. You can ask Noreplyz about an edit if you are very unsure - but generally it is okay if you label the edit and you're not totally sure.

2. I mislabelled an edit

Please open DevTools (CTRL + SHIFT + I), click on the Console tab, and find the diff link that is mislabelled. Then, send a DM to Noreplyz with the requested changes.

3. Can multiple people categorise at once?

Yes - each user is assigned a 'workset' when joining and the workset is unassigned automatically after closing the tab or window.

4. Why are the usernames/timestamps missing?

We want to use this data and train bots to only rely on diffs as an independent datapoint. A user's username or the time of their edit can influence the particular label you apply, so it is hidden.
