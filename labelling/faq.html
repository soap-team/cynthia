Cynthia: diff labelling interface

Instructions

Thank you for helping us with labelling edits. We will use this data to improve and develop our vandalism detection tools.

There are three labels that you can apply to any edit.

  • Good edits should have no labels.
  • Damaging: Does the edit deliberately remove or damage the usefulness of the article content? Any vandalism should be labelled as damaging.
  • Spam: Is the edit an unwanted advertisement?
  • Good-faith: Does it appear as though the author of this edit was trying to contribute productively? Damaging edits may be done with good faith, for example, someone may be attempting to revert vandalism to the wrong version, or they are adding 20 random categories.

The dataset you should work on is en-info-dataset-1. Other datasets may appear, but please focus on this one.

Frequently asked questions

1. I am unsure of an edit

Choose the best labels that describe the edit. You can ask Noreplyz about an edit if you are very unsure - but generally it is okay if you label the edit and you're not totally sure.

2. I mislabelled an edit

Please open DevTools (CTRL + SHIFT + I), click on the Console tab, and find the diff link that is mislabelled. Then, send a DM to Noreplyz with the requested changes. Please provide the workset ID (in the format ws-XX - it should say ws-XX now in use) and the diff link.

3. Can multiple people categorise at once?

Yes - each user is assigned a 'workset' when joining and the workset is unassigned automatically after closing the tab or window.

4. Why are the usernames/timestamps missing?

We want to use this data and train bots to only rely on diffs as an independent datapoint. A user's username or the time of their edit can influence the particular label you apply, so it is hidden.
